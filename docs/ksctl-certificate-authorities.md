# Configure Certificate Authorities
{: .no_toc }

CipherTrust manages one or more internal **local CAs** for use in registering
client authentication and other internal security measures. To use a certificate
generated by a CA external to CipherTrust, it must be added as a trusted
**external CA**.

## External CAs

To integrate CipherTrust with a corporate PKI, the root certificate authority
must be added as a trusted CA. This is required for using an externally signed
cert for the web interface SSL, connecting to Secure LDAP, or generating CTE
client certificates outside of CipherTrust.

### Add External CA

```bash
ksctl ca externals upload --cert-infile ca-thales-lab.crt
```

### Add External Subordinate CA

{: .note }
> To link a new external CA to an existing CA, you will need the `uri` for the
> parent CA.

```bash
ksctl ca externals upload --cert-infile issuer-thales-lab.crt --parent "kylo:kylo:naboo:external_ca:a77ffb32-97b0-467b-9017-5171d7879a39"
```
